# Patch 392B: Retire legacy Coach route

## Summary

Patch 392B retires the legacy `/coach` route as an independent product surface. The route is preserved as a compatibility URL with a server-side redirect to `/guide/pizza-troubleshooting`.

Patch 392A source of truth:

- Audit commit: `91d562a2`
- Audit document: `docs/audits/patch-392a-coach-dependency-audit.md`
- Conclusion: Conclusion B, `/coach` is removable but specified shared modules must remain.

## Patch 392A conclusion applied

Patch 392A confirmed:

- Coach-specific implementation was isolated to `app/coach/page.tsx`, `app/coach/layout.tsx` and `lib/pizza-coach.ts`.
- `buildCoachAdvice` generated pure local text and recommendation output.
- Coach did not own formulas.
- Coach did not mutate shared state.
- Coach outputs were not persisted.
- Coach fields were not part of session schema.
- Coach had no localStorage, cloud, database or account-history dependency.
- Active production links still pointed to `/coach`.
- Shared recipe, flour, saved-recipe, session, account, auth, cloud and Party Order modules must remain.

No Coach advice was migrated into Troubleshooting, Review, Recipe, Session Start or Account history because Patch 392A found no active product dependency.

## Links replaced

| Surface | Old target | New target | Query handling | Rationale |
| --- | --- | --- | --- | --- |
| Shared navigation | `/coach` | removed from navigation | none | Coach is no longer a standalone product concept. Troubleshooting already appears in the Learn group. |
| Homepage secondary tools | `/coach` | `/guide/pizza-troubleshooting` | none | The remaining user job is troubleshooting, not a separate Coach destination. |
| Calculator desktop contextual action | `/coach?${recipeParams(currentSettings)}` | `/guide/pizza-troubleshooting` | old recipe query discarded | Troubleshooting does not support a tested recipe-query handoff. |
| Calculator mobile contextual action | `/coach?${recipeParams(currentSettings)}` | `/guide/pizza-troubleshooting` | old recipe query discarded | Same as desktop; no unsupported handoff was invented. |

Visible Coach label cleanup:

- `Pizza Coach` was removed from active navigation/homepage/calculator surfaces.
- The replacement visible label is `Troubleshooting`.

## Redirect implementation

`app/coach/page.tsx` now contains only a server-side redirect:

```ts
import { permanentRedirect } from "next/navigation";

export default function CoachRedirectPage() {
  permanentRedirect("/guide/pizza-troubleshooting");
}
```

This redirect:

- is handled server-side by Next.js
- does not use `useEffect`
- does not render a loading or compatibility UI
- does not forward unsupported legacy recipe query parameters
- does not create, replace or mutate a Pizza Session

## Advice-engine deletion decision

Decision: delete `lib/pizza-coach.ts`.

Reason:

- `buildCoachAdvice` had no production consumer outside `/coach`.
- Its output was not persisted.
- Its fields were not part of any session schema.
- Its recommendations were not surfaced through Review, Recipe, Troubleshooting or account history.
- Similar user jobs are already covered by canonical learning and troubleshooting routes, but no active dependency required migration.

Potential usefulness was not treated as an active dependency.

## Query compatibility decision

Legacy `/coach?...` URLs redirect to `/guide/pizza-troubleshooting` and discard old query parameters.

Reason:

- Patch 392A found no Coach-specific query parameters.
- Existing `/coach?...` links used shared recipe parameters generated by calculator surfaces.
- Troubleshooting does not currently consume those parameters.
- No hidden recipe-to-troubleshooting handoff was added.
- No Pizza Session is created or replaced.
