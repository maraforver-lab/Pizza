# Kitchen Mode

Patch 36 adds `/session/kitchen`.

Kitchen Mode is the mobile-first step-by-step view for an active local Pizza Session. It is designed for the moment when the user is actually preparing and baking pizza and wants one clear answer:

> What should I do now?

## What Kitchen Mode reads

Kitchen Mode reads the active Pizza Session from the existing local browser storage keys:

```text
doughtools:pizza-sessions-v1
doughtools:active-pizza-session-id
```

It uses:

- the active session id
- the saved session timeline
- timeline step statuses
- the saved `recipeSnapshot` where available
- the user’s experience level
- existing recipe query parameters for links back to tools

It does not upload data to Supabase and does not sync across devices.

## Missing state behavior

Kitchen Mode is safe when session data is incomplete.

- If there is no active session, it links to `/session/start`.
- If there is no timeline, it links to `/session/timeline`.
- If there is no `recipeSnapshot`, it still shows timeline tasks with reduced detail and links to `/session/recipe`.

It does not invent ingredient amounts, fake schedules or unsupported formulas.

## Current task logic

The current Kitchen Mode task is the first timeline step with status:

```text
todo
```

The next task preview is the next later `todo` step after the current task.

If all tasks are done, Kitchen Mode shows a completion state and links to `/session/review` for local bake notes.

## Mark done behavior

The “Mark done” action updates only the local Pizza Session.

It:

- marks the current timeline step as `done`
- preserves the session id
- preserves the recipe snapshot, shopping list and other session fields
- updates `updatedAt` and `lastSavedAt`
- advances `currentStep` to `prep`, `bake` or `review` based on the next timeline task
- keeps the data private to the same browser/device

It does not add push notifications, reminders, cloud sync, account sync, analytics or public sharing.

## Recipe snapshot use

For the Mix dough task, Kitchen Mode can show saved dough numbers from the existing `recipeSnapshot`:

- total dough
- flour
- water
- salt
- yeast / leavener
- dough ball count and ball weight

Kitchen Mode does not recalculate these values. The snapshot is generated by the `/session/recipe` step using the existing calculator helpers.

## Experience-level behavior

Kitchen Mode uses the existing experience-level system:

- Beginner: short, calm instructions.
- Enthusiast: practical why-it-matters notes.
- Pizza Nerd: assumptions, constraints and tradeoffs.

The storage key remains:

```text
doughtools.experienceLevel
```

## Workflow relationship

Current intended guided flow:

1. `/session/start`
2. `/session/recipe`
3. `/session/timeline`
4. `/session/shopping`
5. `/session/kitchen`

Kitchen Mode links back to:

- `/session/timeline`
- `/session/shopping`
- `/session/recipe`
- `/timer` during the Bake pizza step
- `/session/review` during or after Review result
- `/journal` from the review step when the user wants deeper notes

The full calculator remains available and is not replaced.

## What Patch 36 does not implement

Patch 36 does not add:

- new dough formulas
- new planner timing logic
- custom kitchen timers
- reminders
- notifications
- calendar sync
- photo upload
- share cards
- public bake pages
- cloud sync
- account-connected session storage
- analytics or tracking
- Google indexing changes

## Future improvements

Future patches can safely build on Kitchen Mode with:

- richer task-specific visuals
- better oven-specific bake guidance
- optional timer handoff improvements
- session review notes
- result cards
- photo capture
- share-card generation
- optional account sync after a separate privacy and schema review
